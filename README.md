# Эксперимент: реализация iALS на явных данных (MovieLens)

Данный репозиторий содержит рекомендательную систему, построенную на датасете MovieLens 1M. В качестве основного алгоритма используется матричная факторизация iALS из библиотеки `implicit`.

Алгоритм iALS изначально работает с неявным фидбеком. Для его применения на датасете с явными оценками данные были бинаризованы. 

## Логика работы

Подготовка данных начинается с удаления оценок ниже 4. Это оставляет только достоверный позитивный сигнал. Затем пользователи и фильмы с количеством взаимодействий меньше 10 отфильтровываются для сглаживания проблемы холодного старта. 

Для оценки качества применяется time-based валидация для каждого отдельного пользователя. В тестовую выборку отправляются последние по времени взаимодействия (до 5 фильмов, но не более 20% от всей истории пользователя). "Холодные" пользователи и фильмы удаляются из теста.

Подбор гиперпараметров проводился с помощью библиотеки Optuna с оптимизацией по метрике NDCG@10. Найденные параметры зафиксированы в основном скрипте. Масштабирование уверенности (alpha) не применялось, так как эксперименты в ноутбуке показали, что на бинаризованных данных оно приводит к переобучению.

## Структура проекта

* `data/` — директория для датасета.
* `notebooks/main_model_experimenets.ipynb` — анализ данных и поиск параметров через Optuna.
* `src/data_prep.py` — загрузка, фильтрация и сплит датасета.
* `src/model.py` — класс модели.
* `src/metrics.py` — функции расчета NDCG и Recall.
* `src/inference.py` — генерация рекомендаций.
* `main.py` — запуск полного цикла обучения и оценки.

## Результаты

Метрики на тестовой выборке:
* NDCG@10: 0.0828 (8.28%)
* Recall@10: 0.106 (10.60%)

## Инструкция по запуску

1. Скачайте архив MovieLens 1M с сайта GroupLens и распакуйте его в `data/ml-1m/`.
2. Установите зависимости командой `pip install -r requirements.txt`.
3. Выполните `python main.py` для запуска пайплайна.
